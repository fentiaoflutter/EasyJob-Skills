# App Requirement HTML Doc

> 一句话目标：把"全量需求文档"筛选成仅含 APP 实现 / 验收的内容，输出"管理后台风格"中文单文件 HTML + 同源 MD，并维护一份需求文档首页。

## 触发场景

用户这样说时调用：

- 「生成 / 更新需求分析文档 / 技术设计文档 / 任务拆分 / 完成清单 / 自测用例」
- 「一个 HTML 文件包含全部内容」「只要 APP 需求和修改点」
- 「按接口文档持续更新接口/联调部分」

## 生成前约束

1. **必须先确认有需求文档**；无文档先索要。
2. **必须过滤非 APP 信息**（版本日志、评审记录、模板说明等）。
3. **文件名中文**且以需求号开头。
4. **目录约定**：`docs/需求文档/pr-<编号>/` 下放该 PR 文档。
5. **维护首页**：`docs/需求文档/需求文档首页.html` —— 每次新建 / 更新任一 PR 文档**必须同步更新**。
6. **双文件同源**：每次生成 `<中文文件名>.html`（给人看）+ `<中文文件名>.md`（给 Agent 读），同名同目录，禁止只产 HTML。

## 固定页面结构（必须）

左侧导航 + 右侧单页切换（**非一页到底**），中文章节编号：

1. 一、文档概览
2. 二、顶部高亮块（可编辑）
3. 三、需求分析（仅 APP 范围）
4. 四、技术设计（含图）
5. 五、接口 / 联调（独立跟踪）
6. 六、任务拆分
7. 七、完成清单（可勾选可保存）
8. 八、自测用例

每个 PR 文档必须在左侧导航顶部提供"返回需求文档首页"，链接到 `../需求文档首页.html`。

## 各章节硬性要求

### 四、技术设计

- **先放图，再放文字**：
  - 流程图（数据取值 + 兜底计算）
  - UML 时序图（页面加载 → 展示链路）
- **优先内嵌 SVG**（避免 Mermaid CDN 本地失效）。
- 图下方再写数据映射、计算规则、UI 交互、容错、验收映射。

### 六、任务拆分

- **一体化表格**（任务与排期不分离）。
- 列：任务 / 内容 / 相对排期（D1/D2）/ 耗时（可编辑）/ 前置依赖 / 里程碑 / 完成（复选框）。
- 排期用 **D1 / D2** 相对工作日，不写具体日期。
- 表格上方加 **"总天数（可编辑）"**；子任务耗时变化时自动联动重算。
- "完成"勾选状态 → `localStorage`。

### 七、完成清单

- 筛选：`全部 / 前端 / 接口/联调 / 测试`
- 操作：`全部完成 / 全部清空 / 导出进度 / 导入进度`
- 状态持久化：`localStorage`
- 进度显示：已完成数量 + 百分比
- 当前筛选下的"全部完成 / 清空"只作用于当前分组

### 存储 key 规范

- 清单状态：`checklist_<需求号>`
- 高亮块文本：`highlight_<需求号>`

## 执行步骤

1. 解析需求文档，抽取 APP 有效信息：页面 / 模块范围、字段定义与计算规则、精度 / 单位 / 交互提示、验收场景。
2. 输出固定 8 章节内容，绑定左侧导航与页面切换。
3. 技术设计区生成流程图 + UML 时序图（内嵌 SVG）。
4. 完成清单按分组生成并接入 `localStorage`。
5. 预留"五、接口 / 联调"独立章节，后续可增量更新。
6. 放入 `docs/需求文档/pr-<编号>/`，同时生成：
   - `<中文文件名>.html`（主交付）
   - `<中文文件名>.md`（Agent 解析版）
7. `.md` 必须与 HTML 章节语义一致，至少包含：
   - 同样 8 章标题与编号
   - 字段定义 / 公式 / 验收口径
   - 接口联调状态、任务拆分、完成清单、自测用例
   - 明确 TODO / 待确认项（`> [!TODO]` 或 `TODO:` 前缀）
8. 同步更新 `docs/需求文档/需求文档首页.html`（强制）：
   - 新增 / 更新该 PR 的入口卡片
   - 更新"最近更新"日期
   - 链接可直接打开对应 HTML
   - 若有同名 `.md`，同步补 `.md` 入口或标记

## 增量更新（接口文档到来后）

收到接口文档 / 字段变更，**只**更新以下章节，不随意改其他：

1. 五、接口 / 联调
2. 七、完成清单（接口 / 联调任务项）
3. 八、自测用例（受影响用例）
4. 四、技术设计（必要时更新流程 / 时序图）
5. 需求文档首页（更新最近更新时间）

## 输出质量要求

- 全中文，编号连续，导航与正文编号一致。
- 字号偏大，适配评审阅读。
- 左侧导航贴左固定，右侧独立页面切换。
- 公式、单位、精度来源明确，注明"后端字段优先，缺失前端兜底"。
- "等待产品补充 / 待确认 / TODO"统一用**轻量警示样式**：
  - `TODO` 胶囊徽标（浅色，不用高饱和整块底色）
  - 文本块用"左侧警示条"样式
  - 表格中的 TODO 行用浅底高亮
- `.md` 文档面向 Agent 可读性优化：标准 Markdown 标题层级、公式优先用文本表达、字段名与 HTML 一致便于自动比对。

## 禁止事项

- 不许把非 APP 信息（后台 / 评审记录）混进 HTML。
- 不许只产 HTML 不产 MD。
- 不许只更新单个 PR 文档而忘记同步首页。
- 增量更新接口章节时，不要顺手改其他章节。

## 示例

> **用户**：基于这份需求，生成 PR-12345 的需求文档。
> **Skill**：过滤非 APP 内容 → 8 章节 HTML + 同源 MD → 技术设计区放流程图 + UML SVG → 完成清单接 localStorage → 写入 `docs/需求文档/pr-12345/<标题>.html` 与 `.md` → 同步首页卡片 + 最近更新日期。
